EFFET:
──────
Cette modification ajoute une 3ème stratégie qui garantit que même si aucun véhicule
ne peut contenir une réservation ENTIÈREMENT, on cherche au moins 1 place libre pour
assigner minimalement.

Comme le tri est CROISSANT (petites réservations d'abord), cela signifie que:

Exemple: Véhicule 10 places + [8, 3, 1] personnes
┌────────────────────────────────────────────┐
│ Itération 1: Trier [1, 3, 8]               │
│ - Réservation 1: trouvé véhicule → 1 OK   │
│ - Réservation 3: trouvé → 3 OK             │
│ - Réservation 8: assigné entièrement       │
│ Résultat: Véhicule PLEIN 10/10             │
├────────────────────────────────────────────┤
│ Si scenario différent:                     │
│ Itération 1: Trier [1, 3, 8]               │
│ - Réservation 1: cherche best-fit(1)       │
│   → Stratégie 3: cherche au moins 1 place  │
│   → Assigne 1 (complètement)                │
│ - Réservation 3: reçoit prochaine place   │
│   → Jamais fragmentée avant la 1           │
└────────────────────────────────────────────┘

GARANTIE: Les petites réservations sont TOUJOURS fragmentées EN PRIORITÉ
